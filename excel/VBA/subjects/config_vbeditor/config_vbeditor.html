Macro Security Settings

By default, when opening a workbook with macros, Excel will disable them. There is good reason for this, because macros can be executed upon events, such as the opening of a workbook. An unfamiliar workbook may intend to run malicious code upon your system, and has potential to connect to databases and applications.

To be specific, Excel will by default disable macros 'with notification'. This is in the form of a horizontal bar with an 'Enable Content' button, just below the Excel ribbon, and if you trust the content, you can use this to fully enable it.

If you do not see that bar, or wish to customize your settings, you can change the default by doing the following:

  1. Click on File, above the ribbon
  2. Click on Options (bottom-left, you may need to scroll down)
  3. In the left pane of the window that opens, click “Trust Center”
  4. On the right side the window, click “Trust Center Settings"



  5. In the left pane of the next window, click “Macro Settings”
  6. Select the desired macro setting from the upper-right area of the window, then press OK for both open windows

The Developer Tab

For efficiency, and functionality, you should add the 'Developer' tab to your ribbon, by doing the following:

  1. Right-click on a blank space in the Ribbon, at the top of the screen
  2. Click Customize the Ribbon, on the left side of the window that comes up
  3. In the right pane of the new window, add a checkmark to "Developer"
  4. Press OK

The VB Editor

The VB Editor is the VBA coding interface. To open it, press alt+F11, or click on 'Visual Basic', under the Developer tab.

On the right side is the Code Window, for viewing and editing macros. And at the mid to upper left is the Project Explorer, listing the open workbooks, worksheets, and 'modules' of VBA code. If you record a macro, and then open the VB Editor, you will see a folder called 'modules' in the Project Explorer, containing a module which stores that macro's code.

Modules

Modules are objects that group and store VBA code. They are typically where you will store the code for your macros, and can be added through the VB Editor by clicking on Insert (at the top), and then 'module'. This will add a module to the currently selected workbook.

To change the name of a module, see the Properties Explorer, at the bottom-right of the VB Editor. The first cell of its table refers to the name property, and may be edited, subject to the normal rules of object naming (such as no spaces, cannot start with a number). To avoid naming modules with the same name of any macros (which can cause problems), I use the letter “m” as a prefix.
